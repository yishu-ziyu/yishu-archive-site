## PDF 渲染验证与访问控制方案

### 步骤 1：创建必要的目录结构
在项目根目录创建：
```
public/
└── pdfs/
    ├── aigc.pdf
    ├── labor.pdf
    └── regional.pdf
```

### 步骤 2：放置 PDF 文件
将三个 PDF 文件重命名并放入 `public/pdfs/` 目录：
- 你的 AIGC 论文 → `aigc.pdf`
- 你的劳动经济学论文 → `labor.pdf`
- 你的区域经济学论文 → `regional.pdf`

### 步骤 3：验证渲染（本地测试）
运行开发服务器确认 PDF 正确渲染：
```bash
npm run dev
```
访问包含 PDF 的文章页面，验证：
- PDF 加载成功
- 分页控制正常工作
- 缩放功能正常
- 下载链接可用

### 步骤 4：访问权限设置（可选）
如需限制登录用户才能查看 PDF，需修改代码：
1. 在 ArticleDetail.tsx 中添加登录状态检查
2. 未登录用户显示提示或引导登录

### 注意事项
- 确保 PDF 文件名与 constants.ts 中的配置完全一致
- public 目录下的文件会作为静态资源直接服务
- Vercel/Netlify 部署时会自动包含这些文件
